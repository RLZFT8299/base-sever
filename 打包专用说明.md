### **全自动程序打包工具 使用文档**

##### **1. 软件简介**

全自动程序打包工具是一款基于 Python tkinter 开发的图形界面应用程序，旨在简化 Python 脚本的打包过程。它通过调用 PyInstaller，允许用户以交互方式选择脚本、配置参数，并实时查看打包输出，无需记忆复杂的命令行参数。界面采用卡片式设计，操作直观，适合所有 Python 开发者使用。



##### **2. 系统要求**

**操作系统：**Windows / macOS / Linux（需安装 Python 运行环境）

* **源码运行：**打包后的 .exe 文件：仅在 Windows 10 / 11 环境下测试，其他系统请使用源码或自行打包。

Python 版本：3.6 或更高（源码运行必需）

**依赖库：**tkinter（通常随 Python 一起安装）

&nbsp;              pyinstaller（必需，用于执行打包，可通过 pip install pyinstaller 安装）

##### **3. 安装与启动**

###### **3.1 源码运行**

确保 Python 环境已安装，并安装 PyInstaller：



bash

pip install pyinstaller

将 全自动程序打包.py 保存到本地。



在终端中运行：



bash

python 全自动程序打包.py

3.2 使用打包后的 EXE 文件

双击 全自动程序打包.exe 文件（需提前打包生成）。



程序将直接启动，无需 Python 环境，但打包过程中仍需系统已安装 PyInstaller（工具内部会调用）。



##### **4. 主界面布局**

界面主要分为操作区域和输出区域：



顶部配置区：包含脚本选择、选项勾选、图标选择、额外参数输入等。



底部输出区：实时显示打包过程的命令行输出，并提供“开始打包”、“停止打包”、“清空输出”按钮。



状态栏：底部显示当前操作状态（如“就绪”、“正在打包...”等）。



##### **5. 功能模块详细说明**

###### **5.1 选择 Python 脚本**

点击“浏览…”按钮，选择要打包的 .py 文件。



选择后，程序会自动将输出文件夹路径设置为脚本所在目录下的 dist 文件夹。



###### **5.2 打包选项**

生成单个 exe 文件：对应 --onefile 参数，将程序打包成一个独立的 exe 文件（默认勾选）。



隐藏控制台：对应 --windowed 参数，适用于图形界面程序，运行时不显示命令行窗口（默认勾选）。



###### **5.3 图标文件（可选）**

点击“浏览…”选择 .ico 格式的图标文件，该图标将作为打包后 exe 的图标。



若不选择，则使用 PyInstaller 默认图标。



###### **5.4 额外参数（可选）**

可输入额外的 PyInstaller 参数，例如 --hidden-import module、--add-data "src;dst" 等。多个参数用空格分隔。



###### **5.5 输出位置**

显示打包完成后生成的 exe 文件所在文件夹（即脚本目录下的 dist）。



点击“打开文件夹”可直接在资源管理器中打开该目录。



###### **5.6 开始打包**

点击“开始打包”按钮后，程序将执行 PyInstaller 命令，并在下方输出区域实时显示打包日志。



打包过程中可点击“停止打包”终止进程。



打包完成后，若成功则弹出提示框，并刷新输出文件夹路径。



###### **5.7 清空输出**

点击“清空输出”可清除下方输出区域的所有内容，便于查看新的打包日志。



##### **6. 操作提示与注意事项**

必须安装 PyInstaller：如果系统未安装 PyInstaller，启动打包时会弹出错误提示。请提前运行 pip install pyinstaller。



参数安全性：额外参数会直接传递给 PyInstaller，请确保参数格式正确。本工具仅按空格分割参数，若参数中包含空格（如路径），建议直接使用命令行测试。



输出目录：每次打包后，生成的 exe 文件位于脚本所在目录的 dist 文件夹中。如果之前打包过，旧文件会被覆盖。



停止打包：若打包过程卡住或需要取消，可点击“停止打包”强制终止进程。



兼容性：生成的 exe 默认与当前操作系统和 Python 版本相关，如需在其他电脑上运行，建议在目标环境中打包或使用 --onefile 参数。



##### **7. 常见问题**

*Q: 为什么点击“开始打包”后没有反应？*

*A: 请检查是否已安装 PyInstaller。若已安装，查看输出区域是否有错误信息，可能是 Python 脚本路径包含空格或特殊字符，建议将脚本放在简单路径下（如不含空格）。*



*Q: 打包过程中输出乱码或中断？*

*A: 可能是 PyInstaller 版本问题，尝试升级 PyInstaller：pip install --upgrade pyinstaller。*



*Q: 能否为打包后的 exe 添加版本信息？*

*A: 可以使用额外参数添加 --version-file file.txt 或通过 spec 文件配置，本工具未提供直接界面，但可在“额外参数”中手动输入。*



*Q: 图标文件必须是 .ico 格式吗？*

*A: 是的，PyInstaller 要求图标为 Windows 图标格式（.ico）。其他格式需转换后再使用。*



*Q: 工具本身能否打包成独立 exe？*

*A: 可以。使用本工具选择 全自动程序打包.py，勾选“生成单个 exe 文件”和“隐藏控制台”，即可打包出可执行文件，方便分发给其他用户。*
